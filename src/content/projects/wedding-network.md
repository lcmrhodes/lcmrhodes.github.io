WeddingNetwork started as an attempt to make a relational dataset feel personal instead of analytical. The underlying work involved network analysis in R, but the web app had to feel less like a chart and more like an invitation someone would actually want to explore.

The final interaction centers on a guest-specific constellation reveal. That meant balancing a custom visual language, smooth zooming and focus states, and a structure that still respected privacy. The network data shipped to the site stays anonymised, while the frontend preserves the invitation mood rather than flattening the project into a standard graph demo.

The project matters here because it sits directly at the intersection I want this site to represent: visual explanation, product feel, and enough engineering discipline to make the interaction hold together cleanly.
